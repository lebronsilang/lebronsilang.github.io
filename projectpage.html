01 Overview The Why

Started as a school project. Became something personal.

Every finance app I'd used felt clinical — rows of numbers in a utilitarian grid. Tally started as a Software Project Management deliverable, but the concept came from a real frustration with how cold money-tracking tools looked and felt.

Tally combines a visual, Pinterest-style spending journal with real budgeting tools, a grocery tracker, and a dream board that's all wrapped in a themeable, minimal interface. The app development continues past the semester because the problem it solves is real.

02 Features What It Does
01
Feed - Spending Journal
Expenses rendered as cards in an adaptive masonry grid of 2 to 5 columns based on window width. Cards with a photo get a full-bleed image with a gradient overlay. Filter by all time, a specific month, or the current year.
02
Add / Edit Expense
A single page handles both add and edit flows via a query property (ExpenseId). Supports category selection, amount, date, notes, and an optional photo. Categories are a fixed enum: Food, Transport, Shopping, Health, Entertainment, and more.
03
Budget Overview
Set a spending limit per category. Each category card shows a progress bar that turns red when you go over. Edit limits inline without leaving the page.
04
Grocery Tracker
Group-based grocery lists with estimated prices per item. A budget bar shows how much of your grocery budget remains. Check off items as you shop; clear checked items in bulk.
05
Dream Board - Wishlist
A wish list with Priority tags (Need, Want, Someday), a cooling-off reminder system, and a status flow from Planned to Bought. Bought items log a regret rating. Pinned items float to the top. Wishlist items can be converted directly into expenses.
06
Themes
Six presets - Clean Minimal, Sakura Bloom, Mocha Latte, Ocean Mist, Forest Dew, Midnight, and a custom palette picker for background, accent, card, and text colours. Theme changes rebuild the shell immediately so every screen updates at once.

04 Architecture Key Decisions
—
Dependency Injection
Configured in MauiProgram.cs. Services registered as singletons; ViewModels and Pages as transients. This keeps services alive across navigation while pages stay lightweight and garbage-collectable.
—
Theme System
Writes colour values directly into Application.Current.Resources at runtime. When a theme changes, ThemeService.ReloadShell() replaces MainPage with a fresh AppShell so every page picks up the new palette immediately — no page refresh or navigation required.
—
Masonry Grid
Built programmatically in code-behind (FeedPage.xaml.cs, WishlistPage.xaml.cs) — intentionally, because layout construction that depends on measured widths doesn't belong in a ViewModel. The XAML file for those pages contains only a named MasonryGrid container.
—
Shared Add / Edit Page
Add and Edit expense share one page. The route passes an ExpenseId query parameter; IsEditMode on the ViewModel drives the page title, subtitle, and button label — no duplicate code, no duplicate XAML.
